# Skills Overview

Skills are reusable capability packages that extend what AI agents can do on Starknet. Each skill provides focused instructions, concrete install paths, and source-linked references for a specific job.

## What are Skills?

A skill is a self-contained package defined by a `SKILL.md` file with:

- YAML frontmatter for metadata and discovery
- workflow instructions for the agent
- progressive-disclosure links to local references
- optional scripts, examples, and setup notes

<Callout type="info" title="Skill Format">
Skills follow the [AgentSkills specification](/docs/skills/writing-skills) so the same catalog can be consumed by Claude Code, Cursor, Copilot, OpenClaw/MoltBook, and custom runtimes.
</Callout>

## Installing Skills

Skills are designed to be loaded by AI agents at runtime. The agent reads the skill documentation and uses the provided code patterns to accomplish the user task.

### Method 1: Agent Skills CLI

Use the [Skills CLI](https://github.com/vercel-labs/skills) to install the full catalog or a specific skill into your agent context.

```bash
# Install all skills from the repo
npx skills add keep-starknet-strange/starknet-agentic

# Or install a specific skill
npx skills add keep-starknet-strange/starknet-agentic/skills/starknet-wallet
```

### Method 2: Claude Code Marketplace Bundle

```bash
/plugin marketplace add keep-starknet-strange/starknet-agentic
/plugin install starknet-agentic-skills@starknet-agentic-skills
/starknet-agentic-skills:starknet-wallet
```

### Method 3: Direct GitHub Clone

```bash
git clone https://github.com/keep-starknet-strange/starknet-agentic.git
cp -r starknet-agentic/skills ./skills
```

The agent will automatically discover skills in the `skills/` directory.

## Skill File Structure

Each skill follows this structure:

```text
skills/
|
+-- starknet-wallet/
    |-- SKILL.md          # Skill definition (required)
    |-- package.json      # Dependencies (optional)
    |-- scripts/          # Reference implementations
    |   |-- check-balance.ts
    |   +-- transfer.ts
    +-- .env.example      # Environment template
```

## SKILL.md Format

Every skill has a `SKILL.md` file with YAML frontmatter:

```yaml
---
name: starknet-wallet
description: >
  Create and manage Starknet wallets for AI agents.
  Transfer tokens, check balances, and more.
keywords:
  - starknet
  - wallet
  - transfer
  - balance
allowed-tools:
  - Bash
  - Read
  - Write
user-invocable: true
---

# Starknet Wallet Skill

Documentation content here...
```

### Frontmatter Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Lowercase, hyphenated identifier (1-64 chars) |
| `description` | Yes | Multi-line description of capabilities |
| `keywords` | Yes | Array of discovery keywords |
| `allowed-tools` | No | Tools the skill can use |
| `user-invocable` | No | Whether users can invoke directly |

## Using Skills in Agents

When an agent loads a skill, it gains access to:

1. Code examples it can adapt into the target task
2. Tool usage patterns and execution boundaries
3. Error handling and recovery guidance
4. Configuration and prerequisites

<Steps>
<Step number={1} title="Load the Skill">
The agent reads the `SKILL.md` file and follows the workflow and boundaries described there.
</Step>
<Step number={2} title="Install Dependencies">
If the skill requires packages or CLIs, the agent runs the documented install commands.
</Step>
<Step number={3} title="Configure Environment">
Set required environment variables and local config before running live operations.
</Step>
<Step number={4} title="Execute Operations">
The agent uses the skill references and scripts to accomplish the task while staying inside the documented boundaries.
</Step>
</Steps>
