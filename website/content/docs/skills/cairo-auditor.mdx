# Cairo Auditor

The `cairo-auditor` skill is the repo's security-review orchestrator for Cairo and Starknet code. It combines deterministic preflight checks with parallel audit passes and a strict merge/deduplication gate so the final report stays actionable.

### No Cairo project handy?

Create a minimal vulnerable contract locally and audit that file instead:

```bash
TMP_CAIRO_FILE="$(mktemp "${TMPDIR:-/tmp}/cairo-auditor-demo.XXXXXX.cairo")"
cat > "$TMP_CAIRO_FILE" <<'EOF'
#[starknet::contract]
mod VulnerableUpgrade {
    use starknet::ClassHash;
    use starknet::syscalls::replace_class_syscall;

    #[storage]
    struct Storage {}

    #[external(v0)]
    fn upgrade(ref self: ContractState, new_class: ClassHash) {
        replace_class_syscall(new_class).unwrap();
    }
}
EOF
printf 'Demo contract written to %s\n' "$TMP_CAIRO_FILE"
```

Windows users: create the same contents in a local file such as `test_vuln.cairo` with your editor or PowerShell, then run the same prompt against that file path.

```text
# Codex prompt
macOS/Linux: Run cairo-auditor on the temp file path printed above with --file-output.
Output only the final report.
Windows: Run cairo-auditor on .\test_vuln.cairo with --file-output.
Output only the final report.
```

```text
# Claude Code command
macOS/Linux: /starknet-agentic-skills:cairo-auditor <temp-file-path> --file-output
Windows: /starknet-agentic-skills:cairo-auditor .\test_vuln.cairo --file-output
```

This should yield an upgrade-related finding against the ungated `replace_class_syscall` path.

## What LLMs miss

`cairo-auditor` is strong on concrete exploitability. It is not your final authority for:

- multi-transaction economic setups and incentive manipulation
- unstated invariants or specification bugs
- cross-protocol, oracle, or off-chain trust assumptions

Use it to collapse obvious exploit paths early, then finish with tests and human review.

## Audit pipeline

```text
repo / contract
      |
      v
deterministic preflight
      |
      v
4 vector specialists
      |
      +--> deep mode: +1 adversarial specialist
      |
      v
dedupe + false-positive gate
      |
      v
security-review-YYYYMMDD-HHMMSS.md
```

## Deterministic Repo Fixture (Maintainers / Manual Regression)

If you want the stable regression fixture used by this repository, clone `keep-starknet-strange/starknet-agentic` and audit:

```text
skills/cairo-auditor/tests/fixtures/insecure_upgrade_controller/src/lib.cairo
```

This fixture is intentionally vulnerable and should reliably produce three findings: missing access control on upgrade, missing timelock, and missing non-zero class-hash guard.

This path is primarily for manual maintainer regression. Repository CI runs separate validation checks for the skill and report contract rather than asking users to invoke this fixture path directly.

## Modes

| Mode | Use it when |
|------|-------------|
| `default` | Standard repo or contract review before merge |
| `deep` | High-risk changes need adversarial reasoning on top of the default pass |
| `file-targeted` | You want to re-check one or two contracts after a fix |

## What the Skill Does

1. discovers in-scope Cairo files
2. runs deterministic preflight and local benchmark-backed checks
3. spawns vector-specialist audit passes in parallel
4. deduplicates root causes and emits a single report with confidence gating

## Audit pipeline

```text
discover -> preflight -> bundle -> spawn(4+1) -> merge -> report
  files      rules       partitions    agents       dedup    findings
```

Deep mode adds the adversarial specialist on top of the four vector partitions. Default mode stops at the four vector scans.

## Deterministic Local Scan

```bash
python3 skills/cairo-auditor/scripts/quality/audit_local_repo.py \
  --repo-root /path/to/your/cairo-repo \
  --scan-id my-audit
```

This standalone preflight writes `<scan-id>-YYYYMMDD-HHMMSSZ.md` and `<scan-id>-YYYYMMDD-HHMMSSZ.json` artifacts. The `security-review-*.md` naming is used by the full hosted skill flow.

<Callout type="warning" title="Deterministic scans are not full audits">
Deterministic scorecards are regression gates, not proof of full audit sufficiency. Use them to catch drift, then run the full skill for release-critical work.
</Callout>
