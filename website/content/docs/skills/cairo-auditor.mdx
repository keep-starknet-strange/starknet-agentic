# Cairo Auditor

The `cairo-auditor` skill is the repo's security-review orchestrator for Cairo and Starknet code. It combines deterministic preflight checks with parallel audit passes and a strict merge/deduplication gate so the final report stays actionable.

## What LLMs miss

`cairo-auditor` is strong on concrete exploitability. It is not your final authority for:

- multi-transaction economic setups and incentive manipulation
- unstated invariants or specification bugs
- cross-protocol, oracle, or off-chain trust assumptions

Use it to collapse obvious exploit paths early, then finish with tests and human review.

## Audit pipeline

```text
repo / contract
      |
      v
deterministic preflight
      |
      v
4 vector specialists
      |
      +--> deep mode: +1 adversarial specialist
      |
      v
dedupe + false-positive gate
      |
      v
security-review-YYYYMMDD-HHMMSS.md
```

## Modes

| Mode | Use it when |
|------|-------------|
| `default` | Standard repo or contract review before merge |
| `deep` | High-risk changes need adversarial reasoning on top of the default pass |
| `file-targeted` | You want to re-check one or two contracts after a fix |

## What the Skill Does

1. discovers in-scope Cairo files
2. runs deterministic preflight and local benchmark-backed checks
3. spawns vector-specialist audit passes in parallel
4. deduplicates root causes and emits a single report with confidence gating

## Deterministic Local Scan

```bash
python3 skills/cairo-auditor/scripts/quality/audit_local_repo.py \
  --repo-root /path/to/your/cairo-repo \
  --scan-id my-audit
```

This standalone preflight writes `<scan-id>-YYYYMMDD-HHMMSSZ.md` and `<scan-id>-YYYYMMDD-HHMMSSZ.json` artifacts. The `security-review-*.md` naming is used by the full hosted skill flow.

<Callout type="warning" title="Deterministic scans are not full audits">
Deterministic scorecards are regression gates, not proof of full audit sufficiency. Use them to catch drift, then run the full skill for release-critical work.
</Callout>
